    Data storage: var, const, let

    Creating a var stores information in the computer's memory with the name of the variable as a short code. Starting in 2015, JavaScript introduced two new keywords for data storage: const and let.

    const is short for constant and it make prevents the rebinding of values. This means you can't assign a constant the value of 7 and then change it to 20. Using different types of data storage improves the efficiency of a programming language, which is something that has been criticized about JavaScript since it's creation.

    More on const.

    let is like var but block scoped. This is a bit more complicated and we'll discuss it more later in the semester. In theory, all JavaScript programs should be written using only const and let, but var is still acceptable in many cases and will work fine for many of the projects we do in class (plus I would have to rewrite all of my notes...)

    button

    A button element is one of the simplest input elements though it isn't as popular as it once was. Here it is in HTML:

    <button id="sumbit">Click me.</button>

    We can also create one in the console:

    var button = document.createElement("button");
document.body.appendChild(button);
button.id = "submit";
button.innerText = "Click me!";

    There are a lot of ways we can writes scripts to detect when the button is clicked. This is a type of event, and we'll talk more in depth about events next week. For now, let's write a very simple event handler to tell our program the button is clicked.

    button.onclick = function(event) {
	console.log(this);
	console.log(event);
	alert("You clicked me :)");
};

    this refers to the element the event listener is bound to.

    this doesn't always refer to an element, in some cases it refers to funcion scope, but for now this will typically refer to an element bound to an event listener. More on this.

    event is an optional argument for the function that gives you data about the event itself, such as what type of event it was, the element, the position of the click within the element, modifier keys and other info.

    If you want to add an event listener to a HTML element that was not added with JavaScript, you need to get a reference to it first, and in most cases save that reference in a variable.

    var button = document.getElementById("submit");

    console.log

    console.log() is a special JavaScript function that can be used to report things that are happening in the program to the developer console. Like all languages, we need a way to print things to the console to see what works and doesn't it. It's extremely useful and easy to use, but you have to remember to be specific. You can put pretty much anything in the argument parentheses for console.log(), so make sure it's something useful, like an element, a variable, or something else that you want to be able to read. This is a good way to read data from various events.

    input

    There are many different types of input. Today we'll just use a text input:

    <input id="name" type="text" placeholder="Input name">
    var input = document.createElement("input");
input.id = "name";
input.placeholder = "Enter user name";
document.body.insertBefore(input, button);

    We can use JavaScript to grad the value the user types in:

    button.onclick = function() {
	var name = input.value;
};

    A textarea is a resizable input for text.

document.body.insertBefore(email, input);
