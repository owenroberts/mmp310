Although we haven't talked about them specifically, we have been using objects all semester. In fact, basically everything in JavaScript is an object, including arrays, functions, and anything that has methods or properties.

JavaScript is based on an object oriented programming design. This means the language is organized around objects and data rather than logic and actions.

Simply put, and object is a collection of properties and methods associated with one, well, object.

Properties and methods are assigned with key value pairs.

Key, value pairs

In a key value pair, the key is a string and the value can be any JavaScript value, a number, string, array, function or another object.

Single values like numbers, strings and booleans are called properties, while functions are called methods.

For the Array object, .length is a property and .push() is a method.

An array is an object that contains key values for each of the elements in the array as well as other methods and properties.

Trains

Almost any object in the real world can be used as a metaphor for a JavaScript object. Let's use trains.

Like objects in the real world, JavaScript objects have properties and methods, or qualities and actions.

Trains all have one basic blueprint. This is like the object.

A train has many properties, such as color, speed, number of passengers, distance traveled, origin and ending locations, etc.

It also has actions, like functions, it moves on tracks, it honks it's horn, it carries passengers.

Object literal

Let's create a JavaScript version of a train:

var train = {
	speed: 60,
	numberOfPassengers: 500,
	distance: 100,
	origin: "Queens",
	destination: "Brooklyn"
};
console.log(train);

We can set and update properties of the train using dot notation object.property.

// add a new property "color"
train.color = "blue";
// change an existing value
train.numberOfPassengers = 400;
console.log(train);

Object properties can also be referenced with bracket notation:

train["color"];
train["origin"];
train["destination"] = "Manhattan";

So far we only have properties, so let's define a method:

train.getDuration = function() {
	return this.distance / this.speed;
};
train.getDuration();

Object constructor

Another way to create objects is through an object constructor, which is more like a blueprint for an object that get created later.

function Train(color, speed, distance, origin, destination) {
	this.color = color;
	this.speed = speed;
	this.distance = distance;
	this.origin = origin;
	this.destination = destiantion;
	this.getDuration = function() {
		return this.distance / this.speed;
	};
}

new

With an object constructor the new keyword is used when creating a new object.

var cTrain = new Train("blue", 60, 100, "Queens", "Brooklyn");
var twoTrain = new Train("red", 60, 200, "Bronx", "Brooklyn");
