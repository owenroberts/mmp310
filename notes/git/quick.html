If you're creating a new Git repository and GitHub repo for a new project or collaboration you don't need all of the notes from the class overview. Here's the quickest setup for creating a new project.

Git vs GitHub

The first thing to think about is where you want to create the repo. Remember that the repo you have on your local computer and the one you push to GitHub are different copies of the same Git history, so you need to choose one to start with first.

We're going to start with creating a repo locally on your computer first.

Local git repo

Make a folder for your project. You need to know where it is. You can do this using Terminal or the Finder.

First choose where you want the folder to be. I recommend the Desktop. Documents is another good option.

Navigate to your folder with Terminal.

$ cd Desktop
$ mkdir new_project
$ cd new_project

Once you have created the folder and moved into with Terminal we can initialize the repo. This basically sets up the Git system inside your project.

$ git init

You will get a message like Initialized empty Git respository with the path to your project on the computer.

.gitignore

You want to add a .gitignore file to your repo before you make your first commit. The easiest way is to just add and update it in Terminal.

$ touch .gitignore
$ echo '.DS_Store' > .gitignore

That will add a line to ignore all the default Mac files you don't want in your repo. To ignore other things, open the .gitignore and edit it in Sublime or Text Edit.

git status

Use git status a lot. It doesn't do anything, but it shows you what your repo looks like.

$ git status

Right now it should show some red files. That means you have files to add.

Adding files

At this point you may or may not have added new files like index.html, style.css and app.js. If you haven't, go ahead and add some.

Use the add command to add the files.

$ git add .

The dot at the end of the command means add all files.

Type git status again and you should have some green files.

Making a commit

If you have green files and you want to share them or publish them you need to make a commit.

git commit -m "initial commit"

This is your first commit.

You should get some message about the master branch, the commit hash number and create mode.

git status should say On branch master
nothing to commit.

Rinse and repeat

add and commit are the commands you will use the most. If you change your code, you need to add the changes and then commit them.

Don't forget to use -m "commit message"!!!

If you don't you will end up in the dreaded vim. Here's how to get out of vim.

push to GitHub

Now that you have some code you want to share or publish it's time to set up GitHub.

Make a new repo in your GitHub account.

Do NOT select "Initialize this repository with a README". This will result in an error because your local repo and remote repo will have different initial commits. When you add a README GitHub makes the initial commit for you. If you need a README you can make it locally.
