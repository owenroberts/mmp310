For the first assignment, you will be working in pairs. In order for both people in each group to contribute code and content to the project, you will need to set up a repo, clone it, and add your group member as a collabator.

Start by creating a new project on your Desktop and initializing a git repo. You can start adding some files, like index.html.

One person should start. Create a new repo just like we did on the first day of class.

Add a repository name and description.

Copy the remote location of the repo that appears once it is created:

$ git remote add origin <copied-url>

Add the remote to your git repo.

Make your initial commit and push the code to Github.

Now that your repo is set up, you can add your partner as a collaborator. Go to the code repository on your GitHub page and open the settings tab, then click on the Collaborators menu option.

Add your partner by typing their GitHub username into the Add Collaborator field. When you add them, they will recieve and email from GitHub.

After accepting the invitation, your partner should now clone the repo on their own Desktop. Simply copy the URL to the repo and then use the command:

$ git clone <repo-url>

This will create an enclosing folder and download the project and .git files to your Desktop. Now try changing the files, something simple for now, and make a commit and then push to the origin. Your partner should see their repo updated. At that point your partner needs to update their local copy by using git pull:

$ git pull origin master

Each time your or your partner makes an update, you can pull your code using this process.

Conflicts

If your partner pushes some code that has different lines than your and you attempt to pull the code into your local repo, you will get a merge conflict. Git will ask you to resolve the conflicts and then make a new commit. It will show you what lines and files have conflicts. In those files you will see something like this:

<<<<<<<< HEAD
<p>Welcome to my website!</p>
=======
<p>Welcome to my butt!!!</p>
>>>>>>> new_feature

The section that starts with HEAD is what is on your computer. The section that ends with new_feature is what was added by your partner.

You simply need to choose what code to keep and then delete the non-functional lines.

When you do run into a conflict, let me know and we can go over it.
