Functions are blocks of code that can be called repeatedly.

function myFunction() {
	// code block
}

Functions are like variables that reference a series of code statements rather than data.

The parentheses () are used to invoke or call a function.

Functions help organize code by making it easy to repeat common operations, like drawing a circle or displaying text.

When we use the ellipse function, we're actually running about 20 lines of code in several different files that are part of the p5 library. You can see the code here.

The following example takes the bugs example from week 1 and recreates it with functions.

var x = 100; var y = 100; var s = 100; // size var o = 25; // offset function setup() { createCanvas(640, 180); noStroke(); background(51); rectMode(CENTER); bugOne(); x += 200; bugTwo(); x += 200; bugThree(); } function bugOne() { fill('rosybrown'); ellipse(x, y, s); // body fill('black'); ellipse(x + o, y - o, s / 2); // eye ellipse(x - o, y - o, s / 2); // eye rect(x, y + o, s / 2, o / 4); // mouth } function bugTwo() { fill('plum'); ellipse(x, y, s); // body fill('black'); ellipse(x + o, y - o, s / 2); // eye ellipse(x - o, y - o, s / 2); // eye rect(x, y + o, s / 2, o / 4); // mouth } function bugThree() { fill('gold'); ellipse(x, y, s); // body fill('black'); ellipse(x + o, y - o, s / 2); // eye ellipse(x - o, y - o, s / 2); // eye rect(x, y + o, s / 2, o / 4); // mouth }

This makes the organization of my code in the setup function much more clear. I draw a bug, move x, draw another bug, and move x again and draw the final bug.

But, you can see I actually have more code than I had before.

We can further simplify this example by using arguments.

Arguments

A function can also be written with arguments, which are variables that are declared within the parentheses of the function.

Think of a function like a machine that performs a series of actions. The actions are always the same, but the outcome may change depending on the arguments, or parameters.

Think of a function as a blender. A blender is a machine that does one thing: It blends food together.

Depending on what food you put into it, the result will be different.

The ingredients you put in a blender are like arguments.

If I put ice, milk, berries and bananas in a blender, I'll get a banana-berry smoothie.

If I put kale, celery, apples and pineapples in a blender I'll get a green juice.

The only difference between my bugs is the position on the screen and the color of the bug body.

I can create a function that draws a bug but uses different colors by adding arguments.

var x = 100; var y = 100; var s = 100; // size var o = 25; // offset function setup() { createCanvas(640, 180); noStroke(); background(51); rectMode(CENTER); bug('rosybrown'); x += 200; bug('plum'); x += 200; bug('gold'); } function bug(color) { fill(color); ellipse(x, y, s); // body fill('black'); ellipse(x + o, y - o, s / 2); // eye ellipse(x - o, y - o, s / 2); // eye rect(x, y + o, s / 2, o / 4); // mouth }

See how the value that goes into each argument, bug(value), becomes the color of the bug?

The function is defined with the arument color, which is then used to set the fill.

Any attribute can be made into an argument, including color, position or size.

We can add another argument to set the position of each but we have to make sure to get the order right.

We'll add an argument for x, after the color argument.

var y = 100; var s = 100; // size var o = 25; // offset function setup() { createCanvas(640, 180); noStroke(); background(51); rectMode(CENTER); bug('rosybrown', 100); bug('plum', 300); bug('gold', 500); } function bug(color, x) { fill(color); ellipse(x, y, s); // body fill('black'); ellipse(x + o, y - o, s / 2); // eye ellipse(x - o, y - o, s / 2); // eye rect(x, y + o, s / 2, o / 4); // mouth }

Notice that I didn't have to change any of the code in the function, because it already uses the variable x based on the original code, which is now an argument.
